No reinventes la rueda. Aprende a utilizar el poder de miles de desarrolladores en tu propio código.

¿Qué es una biblioteca?

En programación, una biblioteca (o library) es una colección de funciones, clases y recursos ya escritos por alguien más.

  • Resuelven problemas comunes (matemáticas, archivos, red).
  • Ahorran meses de trabajo y evitan errores.

Analogía: La Biblioteca Pública

Colección Privada: Python base son los libros esenciales que ya tienes en casa.

Biblioteca (PyPI): Un repositorio universal con millones de libros especializados elaboradas y validadas por expertos en sus respectivos dominios, disponibles para ser consultadas y utilizadas sin costo alguno.

importar un paquete : equivale a solicitar un préstamo permanente de ese libro, lo incorporas a tu colección local y puedes consultarlo cuantas veces necesites.

Cómo se usan las bibliotecas

En Python, la instrucción import se utiliza para cargar módulos (bibliotecas) que contienen código ya escrito que podemos reutilizar. Al escribir import seguido del nombre de un módulo, le decimos a Python: "carga este módulo y haz que todo su contenido esté disponible para mí".

Por ejemplo: import math

Cuando ejecutamos import math, Python realiza tres pasos internos:

  • 1 Busca el módulo llamado math en el sistema.
  • 2 Lo carga en la memoria del ordenador (si no estaba ya cargado).
  • 3 Crea un objeto llamado math que contiene todas las funciones matemáticas y constantes incluidas en el módulo.
  • 4 Acceso: Para usar cualquier función, debemos usar el prefijo math. (Notación de Punto). Esto le indica a Python que busque específicamente dentro de esa biblioteca.

Manual vs. Bibliotecas

Aquí tienes un ejemplo de cómo buscar la distancia entre dos puntos en un plano cartesiano, haciéndolo a mano vs con bibliotecas.

Cálculo Manual

geometria_cruda.py — VS Code
# Fórmula: √((x2-x1)² + (y2-y1)²)
def distancia(x1, y1, x2, y2):
    dx = x2 - x1
    dy = y2 - y1
    return (dx**2 + dy**2)**0.5

print(distancia(0, 0, 3, 4)) # 5.0

Usando math.dist

pro_dist.py — VS Code 
import math

# Problema solucionado en 1 línea
d = math.dist([0, 0], [3, 4])

print(d) # 5.0

Importación Selectiva: Elige tus herramientas

Puedes cargar toda la biblioteca o extraer solo las funciones específicas que planeas utilizar.

import math

completo.py — VS Code
import math

print(math.sqrt(16))
print(math.pi)
print(math.cos(0))
Ordenado: Sabes de dónde viene cada función.
Largo: Tienes que escribir "math." siempre.

from math import sqrt, pi

directo.py — VS Code
from math import sqrt, pi

print(sqrt(16))
print(pi)
print(sin(0)) # ❌ ERROR: sin no importado
Brevidad: Usas la función directamente.
Riesgo: Podría confundirse con variables o funciones tuyas.

Alias de Importación: Abreviando tu código

A veces las bibliotecas tienen nombres muy largos y queremos abreviar por comodidad. Un ejemplo común ocurre con las herramientas más populares de Python, donde se suele usar un alias estándar para escribir menos código.

Ejemplo Conceptualmente
import nombre_muy_largo as

Antes

viejo.py — VS Code
import nombre_muy_largo

nombre_muy_largo.funcion()
nombre_muy_largo.otra_funcion()

Después

moderno.py — VS Code
import nombre_muy_largo as n

n.funcion()
n.otra_funcion()
